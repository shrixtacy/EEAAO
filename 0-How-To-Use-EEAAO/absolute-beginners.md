### DO NOT DO THESE THINGS (Seriously)

1. **Don't learn multiple programming languages at once** - Pick Python and stick with it for at least 3 months
2. **Don't jump to AI/ML immediately** - You need programming fundamentals first
3. **Don't follow "30-day coding challenges"** - They create false confidence without real skills

**After 1 month:** You can write simple Python programs that solve basic problems
**After 3 months:** You can build a simple project (calculator, to-do list, basic game)
**After 6 months:** You can choose a specialization path with confidence

This isn't a schedule - it's a sequence. Take as long as you need for each step.

- **Build:** Simple calculator program
- **Warning:** Don't move on until you can write this from memory

### Week 3-4: Control Flow

### Beyond Month 1:

## Common Beginner Mistakes

### "I'm Not Smart Enough"
**Reality:** Programming is a skill, not a talent. Everyone struggles at first. The people who succeed are the ones who keep building projects when it gets hard.

### "I Need to Learn Everything"
**Reality:** Professional developers Google things constantly. Focus on understanding concepts, not memorizing syntax.

### "This Tutorial is Too Easy/Hard"
**Reality:** If it's too easy, you're not building enough projects. If it's too hard, you skipped fundamentals.

## Resources to Avoid (Common Traps)

- **Avoid:** "Learn 10 programming languages" content
- **Avoid:** "Build 50 projects in 50 days" challenges  
- **Avoid:** Advanced topics (algorithms, system design) until you can build basic projects
- **Avoid:** Paid courses that promise unrealistic timelines

## When to Ask for Help

- **Good question:** "I'm trying to build X, but I'm stuck on this specific error"
- **Bad question:** "What should I learn next?" (Follow the roadmap)
- **Good question:** "Can you explain why this code works?"
- **Bad question:** "Can you write this code for me?"
